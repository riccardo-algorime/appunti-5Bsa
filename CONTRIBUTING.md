# Come contribuire

Chiunque della classe puo correggere errori, migliorare gli appunti o aggiungere materiale  nuovo.
Non serve installare niente: si fa tutto dal browser su GitHub.

---

## Correggere un errore in un file

1. Vai su [github.com/riccardo-algorime/appunti-5Bsa](https://github.com/riccardo-algorime/appunti-5Bsa)
2. Naviga fino al file da correggere (es. `italiano/pascoli/mega-schema.md`)
3. Clicca l'icona **matita** (in alto a destra nel file)
4. GitHub ti dira: *"You need to fork this repository"* — clicca **Fork this repository**
5. Fai le tue modifiche nell'editor
6. In basso, scrivi una breve descrizione (es. "Corretto nome opera Pascoli")
7. Clicca **Propose changes**
8. Nella pagina successiva clicca **Create pull request**
9. Fatto! La modifica verra revisionata e integrata

## Aggiungere un file nuovo

1. Vai su [github.com/riccardo-algorime/appunti-5Bsa](https://github.com/riccardo-algorime/appunti-5Bsa)
2. Naviga nella cartella giusta (es. `italiano/`)
3. Clicca **Add file** > **Upload files**
4. Trascina i file o clicca per selezionarli
5. Scrivi cosa stai aggiungendo (es. "Appunti lezione Svevo")
6. Clicca **Propose changes**, poi **Create pull request**

### Formato consigliato

- Usa file `.md` (Markdown) per appunti di testo
- Per PDF, immagini o slide: caricali cosi come sono
- Dai nomi chiari ai file (es. `appunti-svevo.md`, non `doc1.md`)

## Segnalare un problema senza modificare

Se preferisci non modificare direttamente, puoi aprire una segnalazione:

1. Vai su [Issues](https://github.com/riccardo-algorime/appunti-5Bsa/issues)
2. Clicca **New issue**
3. Scegli il tipo:
   - **Segnala errore** — per errori nei contenuti
   - **Proponi materiale** — per suggerire aggiunte
4. Compila il modulo e clicca **Submit new issue**

---

## Cos'e una Pull Request?

Quando modifichi un file su GitHub senza avere accesso diretto, GitHub:
1. Crea una **copia** del progetto nel tuo account (fork)
2. Salva le tue modifiche nella copia
3. Propone le modifiche al progetto originale (pull request)
4. Il proprietario revisiona e accetta (o chiede modifiche)

In pratica: proponi una modifica, qualcuno la controlla, e se va bene viene integrata.

## Serve un account GitHub

Se non ce l'hai: vai su [github.com](https://github.com), clicca **Sign up**, e segui i passaggi.
E' gratuito.
